krd changelog

2.1.2

  • Warn and instruct user to sync system time when AWS returns a SignatureExpired error.
  • Support bare IPv6 hostnames in krssh (contributed by @stigtsp).

2.1.1

  • Append :port to hostnames not using port 22.
  • Unset AWS configuration ENV vars that could potentially throw errors if config files are not able to be parsed.

2.1.0

  • Support rsa-sha2-256 and rsa-sha2-512 private key signature algorithms.
  • Support verification of host signatures using rsa-sha2-256 and rsa-sha2-512 signature algorithms.
  • Terminate SSH session when the host public key does not match the Kryptonite pinned key.
  • Add analytics to kr commands (respects disable analytics setting on phone).
  • Add kr debugaws command to test connection to AWS SQS.
  • Fix kr upgrade notice to only show once.
